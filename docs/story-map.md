# Story Map

Follow these in order. The final result should feel like the finished `React Learning` app, but you will understand it because you built every part.

| Story | Build Outcome | Main Concepts |
| --- | --- | --- |
| 01 | Understand the blank Vite app | project setup, npm, Vite, entry point |
| 02 | Render the first product list from local data | JSX, components, props, arrays |
| 03 | Make search, category filter, and sort work | `useState`, events, derived data |
| 04 | Extract reusable product UI | component boundaries, composition |
| 05 | Add product detail pages | React Router, links, URL params |
| 06 | Fetch products from the backend | `useEffect`, loading, error, API layer |
| 07 | Build custom hooks | reusable behavior, hook rules |
| 08 | Add theme switching | Context, provider pattern, CSS variables |
| 09 | Add the cart drawer | Redux Toolkit, global state |
| 10 | Build checkout | forms, controlled inputs, refs, submit flow |
| 11 | Add orders and backend writes | POST requests, async state, data refresh |
| 12 | Polish into the finished app | performance hooks, accessibility, responsive CSS |

## Finished App Checklist

When all stories are done, your app should have:

- Product catalog with search, category filtering, and sorting.
- Product detail route.
- Cart drawer with quantity changes and totals.
- Checkout page that places an order.
- Orders page that reads submitted orders.
- Light/dark theme.
- A small in-app learning page or concept map.
- Clean folder structure: `api`, `app`, `components`, `contexts`, `data`, `features`, `hooks`, `pages`, `utils`.
