# 01 — Blank Vite App Questions

Use these after completing Story 01.

## Questions

1. What problem does Vite solve in a React project?
2. What happens when you run `npm run dev`?
3. Where is the `dev` command defined?
4. What is the difference between `dependencies` and `devDependencies` in `package.json`?
5. Why should you normally not edit `node_modules`?
6. What is the job of `index.html` in a Vite React app?
7. Why does `index.html` contain `<div id="root"></div>`?
8. Which file connects React to that root DOM element?
9. What does `createRoot(document.getElementById('root'))` do?
10. Why does `main.jsx` render `<App />` instead of writing all UI directly there?
11. What is a React component in the simplest possible words?
12. Why does a component name start with a capital letter?
13. What is JSX, and why does it look like HTML?
14. Why do we use `className` in JSX instead of `class`?
15. What is the purpose of `StrictMode` during development?
16. What is hot reload, and how does it help learning?
17. What kind of styles belong in `index.css`?
18. What kind of styles belong in `App.css` at this early stage?
19. If the page is blank, which files would you check first and in what order?
20. In an interview, how would you explain the boot flow from browser URL to rendered React UI?
