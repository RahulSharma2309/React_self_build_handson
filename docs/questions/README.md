# Story Questions

This folder gives you interview-style and self-check questions for every build story.

Use each question file after reading the matching concept note and completing the matching story:

1. Read `docs/concepts/<story>.md`.
2. Build from `docs/stories/<story>.md`.
3. Answer `docs/questions/<story>.md` without looking.
4. Reopen the code and verify your answers.

The goal is not memorizing answers. The goal is being able to explain the code you wrote, debug it, and defend the design choices in an interview.

## How To Practice

- First pass: answer out loud in simple words.
- Second pass: point to the exact file and line where the idea appears.
- Third pass: change something small in the app and predict what will happen.
- Interview pass: explain tradeoffs, not only definitions.
